# Button — Component Rules

**Page:** `docs/components/button.html` | **CSS:** `docs/shared/components.css` | **Version added:** v2.3.0

## Token prefix
`--btn-primary-*`, `--btn-secondary-*`, `--btn-tertiary-*`, `--btn-ghost-*`, `--btn-danger-*`

## CSS classes
- Base: `.btn`
- Variants: `.btn-primary`, `.btn-secondary`, `.btn-tertiary`, `.btn-ghost`, `.btn-danger`
- Sizes: `.btn-lg` (48px), `.btn-md` (40px, default), `.btn-sm` (32px), `.btn-xs` (24px)
- States: `:hover`, `:active`, `:disabled`, `.btn--loading`
- Icon slots: `.btn-icon-left`, `.btn-icon-right`
- Square variant: `.btn-square` (equal width/height, used by Pagination)

## Page sections (h2 IDs)
`basic-button`, `variants`, `sizes`, `states`, `with-icons`, `full-matrix`, `api`

## Rules
- Five variants only: primary, secondary, tertiary, ghost, danger. Do not add new variants without a Figma reference.
- Primary button = strongest emphasis. Never use two Primary buttons side by side.
- `.btn-square` is used by Pagination — do not change its sizing without checking pagination.html.
- The full matrix is generated by inline JS — do not hardcode rows. Edit the JS data arrays instead.
- Focus ring uses `var(--color-heading)` — never hardcode a colour here.
- `.btn-tertiary` has both a default (borderless) and solid (`.btn-tertiary-solid`) variant — both must be maintained.
- Dark mode: Primary inverts (white bg + dark text). Danger stays the same in both modes.

## React
- React class names use BEM: `.btn--primary`, `.btn--md`, `.btn__label` (double hyphen/underscore)
- React file: `react/src/components/Button/Button.jsx`
- Do NOT merge HTML docs class names with React class names — they are intentionally separate
